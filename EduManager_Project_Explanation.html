1. Project Overview

EduManager is a student management system for managing schools, users, students, teachers, courses, attendance, grades, and analytics. The project has two main parts: a backend API that stores and returns data, and a frontend web app that users operate in the browser.

Frontend
Angular screens for login, dashboard, schools, students, teachers, courses, attendance, grades, and analytics.
Backend
FastAPI REST API receives requests, validates data, performs database operations, and returns JSON.
Database
Stores schools, users, student profiles, teacher profiles, courses, attendance, and marks.

10. How To Create Data In The App

Create School

  1. Login as super admin.
  2. Open Schools.
  3. Click add/create school.
  4. Enter school name, city, email, phone, website, plan, and save.

Create Student

  1. Login as super admin or school admin.
  2. Open Students.
  3. Add student details: school, first name, last name, email, phone, grade level, section, guardian details, and enrollment number.
  4. System creates a user account with role student and a linked student profile.

Create Teacher

  1. Login as super admin or school admin.
  2. Open Teachers.
  3. Enter school, name, email, phone, employee ID, department, qualification, specialization, joining date, and salary.
  4. System creates a user account with role teacher and a linked teacher profile.

Create Course

  1. Open Courses.
  2. Enter school, course name, code, teacher, grade level, section, description, and schedule.
  3. Save the course so attendance and grades can use it.

Mark Attendance

  1. Open Attendance.
  2. Select student, course, date, and status: present, absent, late, or excused.
  3. Save records. The backend stores them in the attendance table.

Add Grades

  1. Open Grades.
  2. Select student and course.
  3. Enter exam name, marks obtained, total marks, grade letter, remarks, and date.
  4. Save. The analytics page can calculate average grades from this data.

12. Login And Authentication Flow

  1. User enters email and password on Angular login page.
  2. Angular sends data to POST /api/auth/login.
  3. Backend checks the user email and verifies bcrypt password hash.
  4. If correct, backend creates a JWT token with user email and role.
  5. Frontend stores the token and uses route guards to protect pages.
  6. Role guard controls which page each role can open.

13. How API Is Connected And Data Is Fetched

In this project, Angular does not talk to the database directly. Angular calls the FastAPI backend, and the backend talks to the database. This is the main flow: 

Angular Page Component
    -> calls Angular ApiService/AuthService function
    -> HttpClient sends request to http://localhost:3001/api/...
    -> FastAPI router function receives request
    -> SQLAlchemy queries or inserts database data
    -> Backend returns JSON response
    -> Angular subscribe() receives JSON
    -> Component variable updates and HTML table/card displays data

API Base URL

The frontend API connection is stored in frontend/src/environments/environment.ts. This file tells Angular where the backend is running. 

export const environment = {
  production: false,
  apiUrl: 'http://localhost:3001/api',
};

If backend is running on another port, change this apiUrl. For this project, backend port is 3001, so frontend calls URLs like http://localhost:3001/api/students.

Main API Service File

Most API functions are written in frontend/src/app/core/api.service.ts. This file uses Angular HttpClient to call backend APIs. 

private base = environment.apiUrl;

getStudents(params?: any): Observable<any> {
  let httpParams = new HttpParams();
  // query params are added here
  return this.http.get(`${this.base}/students`, { params: httpParams });
}

createStudent(data: any): Observable<any> {
  return this.http.post(`${this.base}/students`, data);
}

Meaning: getStudents() fetches students using GET request, and createStudent() creates a new student using POST request.

Where Function Is Called In Frontend

Example: in frontend/src/app/pages/students/students.component.ts, the load() function calls this.api.getStudents(params). When the response comes, it stores data into this.students. 

load() {
  const params: any = { search: this.search || undefined, page: this.pagination.page };

  this.api.getStudents(params).subscribe({
    next: r => {
      this.students = r.students || [];
      this.pagination = r.pagination || this.pagination;
    },
    error: () => {},
  });
}

Then the HTML table uses *ngFor="let s of students" to display all student rows. 

<tr *ngFor="let s of students">
  <td>{{ s.firstName }} {{ s.lastName }}</td>
  <td>{{ s.enrollmentNumber }}</td>
  <td>{{ s.section }}</td>
</tr>

How Create Student Works

When user clicks save in the student form, onSave() runs. If editing is false, it calls createStudent(). After successful save, it hides the form and calls load() again to refresh table data. 

onSave() {
  const obs = this.editing
    ? this.api.updateStudent(this.editing.id, this.form)
    : this.api.createStudent(this.form);

  obs.subscribe({
    next: () => {
      this.showForm = false;
      this.load();
    },
    error: () => {},
  });
}

Backend Function That Receives Student API

The backend student APIs are in backend/routers/students.py. The route prefix is /api/students. So this function handles GET /api/students. 

@router.get("")
def list_students(..., db: Session = Depends(get_db)):
    query = db.query(models.Student)
    students = query.offset(offset).limit(limit).all()
    return {
        "students": [_student_to_dict(s) for s in students],
        "pagination": {
            "page": page,
            "totalPages": total_pages,
            "total": total
        }
    }

This function reads from the students table using SQLAlchemy and returns JSON.

Backend Function That Creates Student

This function handles POST /api/students. It creates a user first, then creates a student profile linked with that user. 

@router.post("", status_code=201)
def create_student(data: schemas.StudentCreate, db: Session = Depends(get_db)):
    user = models.User(

    db.add(student)
    db.commit()
    db.refresh(student)
    return _student_to_dict(student)

14. Login API Code Flow

Login starts from frontend/src/app/pages/login/login.component.ts. When user submits the form, onLogin() calls this.auth.login(email, password). 

onLogin(): void {
  this.auth.login(this.email, this.password).subscribe({
    next: () => this.router.navigate(['/dashboard']),
    error: (err) => this.error = 'Login failed.'
  });
}

The login API function is in frontend/src/app/core/auth.service.ts. 

login(email: string, password: string): Observable<LoginResponse> {
  return this.http.post<LoginResponse>(`${this.apiUrl}/auth/login`, { email, password }).pipe(
    tap(res => {
      localStorage.setItem('sms_token', res.accessToken);
      localStorage.setItem('sms_user', JSON.stringify(res.user));
      this.currentUserSubject.next(res.user);
    })
  );
}

Backend receives this in backend/routers/auth.py at POST /api/auth/login. It checks email, verifies password, creates JWT token, and returns user data. 

@router.post("/login")
def login(request: schemas.LoginRequest, db: Session = Depends(get_db)):
    user = db.query(models.User).filter(models.User.email == request.email).first()
    if not user:
        raise HTTPException(status_code=401, detail="Invalid credentials")

    if not auth_utils.verify_password(request.password, user.password_hash):
        raise HTTPException(status_code=401, detail="Invalid credentials")

    access_token = auth_utils.create_access_token(
        data={"sub": user.email, "role": user.role}
    )
    return { "accessToken": access_token, "user": {...} }

16. How To Create Similar Code Yourself

  1. Create database model in models.py, for example Student.
  2. Create request/response schema in schemas.py, for example StudentCreate.
  3. Create backend router function in routers/students.py, for example @router.post("").
  4. Include router in main.py using app.include_router(students.router).
  5. Create Angular service function in api.service.ts, for example createStudent(data).
  6. Call service function from component, for example StudentsComponent.onSave().
  7. Bind HTML form fields using [(ngModel)] so user input goes into form object.
  8. Use subscribe() to receive response and refresh the page data.
// Simple frontend create pattern
form = { name: '', email: '' };

onSave() {
  this.api.createSomething(this.form).subscribe({
    next: () => this.load(),
    error: () => alert('Error saving data')
  });
}

// Simple backend create pattern

def create_item(data: ItemCreate, db: Session = Depends(get_db)):
    item = models.Item(**data.model_dump())
    db.add(item)
    db.commit()
    db.refresh(item)
    return item

17. How To Explain API Fetching To Sir

"Sir, the frontend and backend are connected through REST APIs. I stored the backend URL in the Angular environment file. Then I created an ApiService file where all HTTP functions are written. For example, when I open the Students page, the component's load function calls getStudents from ApiService. ApiService sends a GET request to /api/students. FastAPI receives it in students.py, queries the database using SQLAlchemy, and returns JSON. Angular receives that JSON inside subscribe and assigns it to the students array. The HTML table uses ngFor to display the array. For create operations, the form data is sent by POST request, backend inserts it into database, and frontend reloads the list."
